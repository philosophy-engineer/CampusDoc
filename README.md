# HCI 줄 단위 읽기 보조

줄 단위 읽기 보조 인터페이스를 비교 평가하기 위한 HCI 수업 프로젝트입니다. 정적 웹 애플리케이션 형태로 구현되어 있으며, 일반 읽기 화면과 유저 스터디 화면, 결과 기록 및 CSV 다운로드 기능을 함께 제공합니다.

## 프로젝트 개요

이 프로젝트는 긴 텍스트를 읽을 때 사용자가 현재 읽고 있는 줄을 더 안정적으로 추적하고, 필요한 정보를 다시 찾는 과정을 보조할 수 있는지 확인하기 위해 만들어졌습니다. 발표 자료와 실제 구현을 기준으로, 다음과 같은 가설을 검증하는 방향으로 설계되어 있습니다.

- 현재 읽는 줄을 시각적으로 드러내는 인터페이스는 기본 읽기 방식보다 줄 이탈을 줄이고 재탐색 효율을 높일 수 있다.
- 줄 보조 인터페이스는 기본 읽기 방식보다 내용 이해도와 문제 해결 성과에 긍정적인 영향을 줄 수 있다.
- 서로 다른 줄 보조 방식은 자연스러움, 부담감, 피로도, 재탐색 용이성 측면에서 서로 다른 사용자 경험을 만들 수 있다.

프로젝트는 별도 빌드 과정 없이 `index.html`, `app.js`, `files/*` 데이터만으로 동작합니다.

## 핵심 기능

### 1. 기본 읽기 모드

- `manifest.json`에 등록된 텍스트 파일 목록을 브라우저에서 탐색할 수 있습니다.
- 개별 텍스트를 읽기 전용 화면에서 열 수 있습니다.
- 라이트/다크 테마를 전환할 수 있습니다.

### 2. 유저 스터디 모드

- 참가자 ID와 그룹(1~4)을 선택해 세션을 시작합니다.
- 각 그룹은 서로 다른 조건 순서로 4개의 실험 단계를 수행합니다.
- 각 단계는 `튜토리얼 -> 본 읽기 -> 퀴즈 -> Likert 설문` 순서로 진행됩니다.
- 모든 단계를 마치면 최종 설문을 작성하고 세션이 완료됩니다.

### 3. 결과 기록 및 CSV 다운로드

- 완료된 세션은 브라우저 `localStorage`에 누적 저장됩니다.
- 기록 화면에서 개별 세션 CSV 또는 전체 세션 CSV를 다운로드할 수 있습니다.
- 완료된 세션만 기록 목록에 누적됩니다.

### 4. 테마 전환

- 앱 전역에서 라이트/다크 테마를 전환할 수 있습니다.
- 현재 테마 선택은 브라우저 저장소에 유지됩니다.

## 인터페이스 조건

유저 스터디는 네 가지 읽기 조건을 비교합니다.

### A. 기본 읽기

- 별도의 줄 보조 장치가 없는 기본 읽기 인터페이스입니다.

### B. 밑줄 안내

- 마우스를 움직이면 현재 줄 아래에 밑줄이 표시됩니다.
- 사용자가 어느 줄을 따라 읽고 있는지 시각적으로 드러내는 방식입니다.

### C. 줄 강조

- 마우스를 움직이면 현재 줄 전체가 강조됩니다.
- 현재 시선이 머무는 줄을 더 넓은 영역으로 인지하도록 돕습니다.

### D. 키보드 이동

- 위/아래 방향키로 줄 단위 이동을 수행합니다.
- 마우스 휠 스크롤은 제한되며, 키보드 기반으로 읽기 흐름을 제어합니다.

## 유저 스터디 절차

### 세션 시작

- 시작 화면에서 참가자 ID를 입력합니다.
- 그룹 1~4 중 하나를 선택하면 세션이 생성됩니다.
- 그룹마다 조건과 본문 텍스트 순서가 다르게 배치되어 순서 효과를 줄이도록 구성되어 있습니다.

### 단계 구성

각 실험 단계는 아래 순서로 진행됩니다.

1. 조건별 튜토리얼
2. 본문 읽기
3. 객관식 퀴즈 풀이
4. 인터페이스별 Likert 설문

네 개의 조건을 모두 마친 뒤에는 최종 설문으로 이동합니다.

### 연습 지문과 본 실험 지문

- `T0`: 튜토리얼용 연습 지문
  - `서랍에-저녁을-넣어-두었다.txt`
- `T1`~`T4`: 본 실험 지문
  - `수면과-기억-공고화.txt`
  - `표준시-제도의-역사.txt`
  - `꿀벌-수분과-생태계.txt`
  - `도시-열섬-현상.txt`

그룹별 조건/지문 순서는 다음과 같습니다.

| 그룹 | 1단계 | 2단계 | 3단계 | 4단계 |
| --- | --- | --- | --- | --- |
| 1 | A + T1 | B + T2 | D + T4 | C + T3 |
| 2 | B + T3 | C + T4 | A + T2 | D + T1 |
| 3 | C + T1 | D + T2 | B + T4 | A + T3 |
| 4 | D + T3 | A + T4 | C + T2 | B + T1 |

## 수집 데이터와 평가 지표

발표 자료와 실제 구현 기준으로, 앱은 다음 정보를 수집하고 세션 완료 시 CSV 한 행으로 정리합니다.

### 읽기 단계

- 읽기 시간
- 마우스 아래 방향 이동 횟수
- 마우스 위 방향 이동 횟수
- 위 방향키 입력 횟수
- 아래 방향키 입력 횟수

### 퀴즈 단계

- 문제 수
- 정답 수
- 정답률
- 문제 풀이 총 시간
- 본문 재방문 횟수
- 첫 재방문 시점부터 제출까지 걸린 시간
- 문항별 퀴즈 화면 체류 시간
- 문항별 본문 화면 체류 시간
- 문항별 본문 재방문 여부 및 횟수
- 문항별 답안 변경 횟수

### Likert 설문 단계

각 조건마다 아래 6개 문항의 점수와 이유를 기록합니다.

- 이 인터페이스는 줄을 놓치지 않도록 도와주었다.
- 이 인터페이스는 내용 이해에 도움이 되었다.
- 이 인터페이스는 원하는 위치를 다시 찾기 쉽게 해주었다.
- 이 인터페이스는 읽기 흐름이 자연스러웠다.
- 이 인터페이스를 조작하는 것은 부담스러웠다.
- 긴 문서를 읽을 때 이 인터페이스를 사용할 의향이 있다.

### 최종 설문 단계

- 가장 자연스러웠던 조건
- 마우스를 따라 읽는 방식의 부담감에 대한 서술형 응답
- 방향키 이동이 안정감을 주었는지 또는 흐름을 끊었는지에 대한 서술형 응답
- 네 조건의 전체 선호 순위와 이유
- 가장 덜 피로했던 조건
- 정보를 다시 찾기 가장 쉬웠던 조건

## 실행 방법

이 프로젝트는 별도 패키지 설치가 필요 없는 정적 웹 앱입니다. 다만 텍스트와 퀴즈 파일을 `fetch()`로 불러오기 때문에 로컬 HTTP 서버에서 실행해야 합니다.

```bash
cd /path/to/hci2
python3 -m http.server 8000
```

브라우저에서 아래 주소로 접속하면 됩니다.

- [http://localhost:8000](http://localhost:8000)

주의:

- `file://`로 `index.html`을 직접 열면 `manifest.json`, 텍스트 파일, 퀴즈 파일 로딩이 정상 동작하지 않을 수 있습니다.

## 프로젝트 구조

- `index.html`
  - 전체 UI 마크업과 스타일 정의
- `app.js`
  - 해시 라우팅
  - 조건별 읽기 인터랙션
  - 유저 스터디 흐름 제어
  - `localStorage` 기반 세션/기록 저장
  - CSV 다운로드 생성
- `files/manifest.json`
  - 브라우저에서 노출 가능한 `.txt` 파일 목록
- `files/hci-reading-quizzes.json`
  - 지문별 객관식 문항 데이터
- `files/*.txt`
  - 튜토리얼 및 본 실험용 텍스트 지문

## 문서화된 인터페이스/계약

### 해시 라우트

- `#/start`: 시작 화면
- `#/browse`: 텍스트 목록 화면
- `#/study`: 진행 중인 유저 스터디 화면
- `#/records`: 완료 세션 기록 화면
- `#/file/:name`: 개별 텍스트 읽기 화면

### 데이터 파일 계약

#### `files/manifest.json`

- `files` 배열에 표시 가능한 `.txt` 파일명을 담습니다.
- 앱은 이 목록을 기준으로 읽기 화면의 파일 목록을 구성합니다.

#### `files/hci-reading-quizzes.json`

- 텍스트별 퀴즈 세트를 제공합니다.
- 각 항목은 `source_file`, `title`, `questions`를 포함합니다.
- 각 문항은 `id`, `task_type`, `prompt`, `options`, `correct_option_id`를 가집니다.

### 브라우저 저장소 키

- `hci_reader_condition`
  - 기본 읽기 화면에서 사용할 현재 조건
- `hci_reader_theme`
  - 현재 테마
- `hci_study_session_v1`
  - 진행 중인 세션
- `hci_study_records_v1`
  - 완료된 세션 기록 목록
- `hci_study_records_meta_v1`
  - 전체 다운로드 횟수, 마지막 전체 다운로드 시각 등 기록 메타 정보

### 출력 형식

- 완료된 각 세션은 wide-format CSV 한 행으로 내보낼 수 있습니다.
- 한 행에는 참가자/그룹/단계별 읽기 기록/퀴즈 결과/Likert 응답/최종 설문 응답이 함께 포함됩니다.

## 구현 메모와 한계

- 백엔드나 데이터베이스 없이 브라우저 `localStorage`만 사용합니다.
- 따라서 결과 데이터는 브라우저별, 기기별로 분리 저장됩니다.
- 저장된 세션은 사용자가 직접 CSV로 내려받기 전까지 로컬 환경에만 남아 있습니다.
- 기록 화면에는 완료된 세션만 누적됩니다.
- 발표 자료에 포함된 시각 예시 이미지나 영상은 이 저장소 안에 앱 문서용 자산으로 정리되어 있지 않으므로, README는 텍스트 설명 중심으로 작성했습니다.
- 이 README는 현재 구현 상태를 설명하는 문서이며, 실제 사용자 연구의 통계 분석 결과나 결론은 포함하지 않습니다.

## 유지보수 메모

- 새 텍스트를 추가하려면 `files/manifest.json`에 파일명을 등록해야 합니다.
- 유저 스터디용 지문을 추가하거나 교체할 경우 `TEXT_FILES`, 그룹 순서, 퀴즈 데이터의 연결 관계를 함께 수정해야 합니다.
- 퀴즈 데이터와 실제 지문 파일명이 어긋나면 스터디 단계에서 퀴즈를 불러오지 못합니다.
